Role Definitions

Oligo enforces role-based access control (RBAC) with two roles available to customer administrators. Access is governed by least privilege principles, as documented in Oligo's Access Control Policy and Information Security Policy.

Admin
  • Full access to manage resources and members across the Oligo environment
  • Can assign and modify roles for other users
  • Can configure Slack and Jira integrations and manage project membership
  • Can select the Default (all) view to see all resources across the organization
  • Can create, modify, and remove projects and associated resources
View-Only
  • Read access to resources within assigned projects only
  • Cannot modify settings, integrations, roles, or project membership
  • Users assigned to two or more projects use the project context selector to switch views
  • Cannot access resources outside of their assigned project membership

Admin Account Lifecycle

Account Setup
Administrators can create new user accounts via the Oligo UI or through SSO/SCIM integration. During setup, the provisioning administrator assigns the appropriate role in line with the user's responsibilities.
MFA Requirements
Multi-factor authentication is enforced at the platform level via Palantir's FedRAMP-authorized infrastructure. MFA is managed through Palantir's Control Panel.
Authentication and Access Controls documentation
Config Checklist
After creating an account, the provisioning administrator should:
  • Assign the correct role based on the user's responsibilities
  • Add the user to relevant projects only (apply least privilege)
  • Confirm integrations are scoped correctly before granting Admin access
Decommissioning
When a user no longer requires access:
Method Steps
With SCIM Deactivate the user in the Identity Provider (IDP). The user is automatically deactivated in Oligo.
Without SCIM Navigate to Settings, click on the user's role, and change it to Viewer. Admin privileges are removed immediately.
Access revocation required within 48 hours of request
